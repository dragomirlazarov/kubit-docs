Query is a very flexible chart designed to help you understand user interactions within your product. It answers questions like "How many users are performing specific events?"

Watch our [Query Training Video](https://help.kubit.ai/en/articles/8672065-query-training)

# Query Use Cases

The Query report is beneficial for various scenarios:

- Determine many users performed an event on my product over the last 7 days.
- Calculate the percentage of paying users who performed our critical event last month.
- Track and understand what our monthly active user count is for the past 6 months.
- Find many users do we have on average over the past 7 days performing any event.

![](/images/docs/21f9d96922dc23dcbc5e235afa429f8ac5d20b6765284e7a0250e90f542102fb-query_use_cases_1.png)

Executing a Query report, like other reports in Kubit, is a read-only operation that doesn't modify your data warehouse.

The default mode for Query reports focuses on "how many" type questions. Additionally, you can explore [Impact Analysis](doc:advanced-query-features#impact-analysis) and the [Histogram](doc:advanced-query-features#histogram) modes. You can try these advanced features to help you identify more use cases to analyze your data.

# Building Your Measures

A measure consists of three components: the event to analyze, the function to compute, and any filters to apply. You can learn more about Measures in this [article about components of a Measure](doc:components-of-a-report#measures).

## Event

Select the event(s) you want to analyze inside of your chart by selecting them through the dropdown.

- You can select multiple events to perform an OR logic, which means all members of the measure had to perform Event 1 or Event 2.
- You can also select any [Virtual Events](doc:components-of-a-report#virtual-events) you may have created. You will see those available at the top of your Event dropdown list.

## Functions

- **Count Events**: This function will count the total number of times the event you've selected has been performed. If a user does an event multiple times, each occurrence of that event will be counted.
- **Uniques**: Counts unique identifiers (e.g., User ID, Session ID) for the selected event.
  - These can be customized to your data model, if you'd like to configure additional functions please reach out to us!
  - Examples include User ID, Anonymous Visitor ID, Session ID, Account ID
- **Math Functions**: This includes Sum, Avg, Min, Max functions that can be applied to numeric event fields.
  - These are only available for field values that are numeric (i.e. 1,2,3)

### Add Additional Events and Measures

When building a Query multiple events within a single measure could be considered part of your metric. In Kubit you are able to achieve this in two ways:

- Add additional events within the same Event module to perform an OR statement. All filters applied will be included for each event.
- Add additional events with the "+ Add Measure".
- Adding additional measures without using the [Formula](doc:query#compound-measures) will result in just the two measures shown on the report visualization.

![](/images/docs/554ac766cf07520d910d64ad03b2dfbc7ecf477e72256e155e98ef10cfdf9123-add_additional_events_and_measures.gif)

# Compound Measures

A Compound Measure is a type of Measure that involves arithmetic between two or more simple Measures like those explained above. An example of a common compound measure is something like 'average events per user'. When you need to calculate as a numerator divided by a denominator, that will be a Compound Measure.

To save a Measure you've created, follow the instructions [here](https://docs.kubit.ai/docs/components-of-a-report#saving-and-updating-measures).

Kubit supports Compound Measures using the Formula bar below the Measure builder.

![](/images/docs/5cce862639d4f369854518300b33a3a561aec6a3da5e21ef74e44b6146053854-Compound_measure.gif)

## Formula Operators

Supported operators will be +, -, /, \*

- (+) addition
  - `A + B`
- (-) subtraction
  - `A - B`
- (/) division
  - `A / B`
- (\*) multiplication
  - `A * B`

In any mathematical expression, it is essential to understand the order of operations to get the desired result. In Kubit, there are 4 operators available. Whenever a division operator is present, Kubit will first evaluate the numerator followed by the denominator and then perform the division operation.

| 1st         | 2nd       | 3rd            | 4th      | 5th      | 6th         |
| ----------- | --------- | -------------- | -------- | -------- | ----------- |
| Parenthesis | Exponents | Multiplication | Division | Addition | Subtraction |

Here are a few examples to illustrate how Kubit calculates a Compound Measure:

- `(4 x 6) ÷ (2) = 12`
- `(4 + 6) ÷ (2 + 3) = 2`
- `(5) ÷ (4 - 1) = 1.6`
- `5 + 1 + 2 = 8`

## Supported Functions

- **min**(A, B) - returns the min value from both measures
- **max**(A, B) - returns the max value from both measures
- **ifnull**(A, \<default\>) - returns A if not null, otherwise the default value
- **sum**(A, B, C) - shortcut for sum of multiple measures
- **ceil**(A, \<scale\>) - returns values from A rounded to the nearest equal or larger integer, or to the nearest equal or larger value with the specified number of places after the decimal point
- **round**(A, \<scale\>) - Returns rounded values for A
- **floor**(A, \<scale\>) - returns values from A rounded to the nearest equal or smaller integer, or to the nearest equal or smaller value with the specified number of places after the decimal point
- **pow**(A, x) - returns measure value raised to the specified power (x)
- **sqrt**(A) - square root of A
- **log**(\<base\>, A) - returns the logarithm of a number given a base
  - The base should be positive number and not equal to 1
  - To solve for an error related to this be sure to set a [Measure Filter](doc:advanced-query-features#measure-filters) where the value must be \> 0

### Example Formulas

- `min(A, B) / C` - is the minimum value between measure `A` and `B`, divided by measure `C`.
- `pow(A, 2)` - `A` raised to the power of `2`.
- `ifnull(A, 1.5)` - if `A` is `NULL`, then return value is `1.5`.
- `ceil`, `round`, and `floor` are utilities for formatting the result measure. For example, `round(A, 3)` rounds the value of the `A` rounds to three decimal places.
- `sum(A, B, C)` is a shortcut for `A + B + C`.
- `ceil(min(A, ifnull(B, 3.1)), 5)` - functions can be nested; round the result to 5th number of decimal places, where the result is the minimum value between `A` and `B` (where if `B` is `NULL`, then it will be `3.1`).

<Info>
  ### Kubit will perform a syntax check

  Once you've written your formula and click out of the formula bar, Kubit will check if the syntax is correct. If the syntax is invalid. you will see an error message helping you fix the formula.
</Info>

# Filter

Once you've selected your event(s) you can filter them depending on the specific use case or analysis you're building.

- Build your own filter through the available operators and utilizing the AND/OR logic if needed.
- If you don't see a field available in the dropdown but may see it on other events that's alright. It means the field you're looking for may not be available within that event.
- You can also select any [Saved Filters](doc:advanced-query-features#measure-filters) you may have created. You will see those available at the top of your field dropdown list.

## Cohort Measure Filter

Often it's important to filter an event within a compound measure by members in a Cohort instead of just an event. To add a Cohort Filter to a Measure you will click the "Measure Options" dropdown and select "Add Cohort Filter".

**Create a New One**: You are able to build the Cohort within the Query builder by clicking "Create Custom"

![](/images/docs/293f96ac96ac2d7e9b41b31cc92f9c9cc1112067bd764049cc80d7dc1d1b9c74-cohort_measure_filter.gif)

**Use an Existing Cohort**: If you've already created a Cohort that you want to use you'll simply select it from the list of available Cohorts.

![](/images/docs/78bf21c5a3cd1be86f8b770f819507a4d9f36b51a60ec7640d9a7d58767f113b-using_an_exisiting_cohort.gif)

# Breakdown

When you're analyzing data it's important to be able to slice it by various attributes of the User or Event. Breakdowns allow you to group the results by these fields and even Bin them into more meaningful groups in Query.

- Breakdown by a Field: Select the field you want to breakdown by to see results grouped based on that fields values. These can be things like "User Type" or "Subscription Mode".

## Additional Bin Options

- **Breakdown with Binning**
  - When you'd prefer to group values inside of the breakdown into groups or "bins" we've made that available in the Breakdown feature.
  - Select the field you want to breakdown by and click the context menu to expose "Add Binning"
    - You can bin items based on their values, Value Binning, or ranges of integers, Range Binning. Based on the data type you've selected you may see only one or both.
  - Once you've created your bin you can save it for Reuse.

  ![](/images/docs/1fcd47cd2f8c81be828a0242afe935d9e2c3ef5435ff8961b21908b5d96d48cf-breakdown_with_binning.gif)

# Compare Mode in Query

Want to compare how a metric has changed over time? You can leverage the Compare feature in Query to achieve that!

![](/images/docs/d321cdf0a4db80e4f2b4541319f3fdc45e47ec043131d67b56f40f05b44ae3a3-compare_mode_in_query.png)

There are multiple options you can leverage from the Compare dropdown:

- Previous `Week/Month/Quarter/Year` - shortcuts to select a previous period based on the current report's date range.
- `Custom` - which allows you to manipulate the end date for the previous period and the start date is adjusted based on the length of the current report's date range

When using `Compare` any metrics for the previous period will be displayed as a dotted line on the chart, while the metrics for the current period will be displayed as a regular line, as shown here:

![](/images/docs/dcb70236776b591a61aedabfcc1b54ab62cb99752b07b7bbb97d8d4575b86b52-compare_dotted_line.png)

But even better, when you switch the Query result from Line Chart to Numeric you will also see the % change and that will make your Dashboards even more awesome:

![](/images/docs/82fc91aaaded50beea020da2dbcfb9a0ad179978edb70c7c45b64b94802a1c1d-line_chart_to_numericmetric.gif)

## Report Visualizations in Query

When you create a Query report you also get access to additional chart types:

- Stacked Column
- Percentage Area
- Pie Chart
- Combo Chart
- Metric

![](/images/docs/4190aaf39faf1ca9917276da7406a22694ede666177d984c43ce8573bbc7e294-report_visualizations_in_query.gif)

To access the Combo Chart (combination of a Line and Bar chart) you need to:

- Select "Combo" from the chart list.
- Click on the cog icon to adjust chart settings indicating which measure should be a line or bar chart.
- Click "Save" in the upper right corner to save your changes.

# Format Report Options

Query offers several customizations to how you can visualize your results. All report settings can be accessed by clicking the cog icon in the top right of the report visualization.

## Measure Assignments

All measures on a Query report will be listed here. From this menu you're able to:

- Show or hide these Measures by clicking the Measure name
- Assign them to a Left or Right axis
- When in Combo Chart mode you're able to determine which Measure is a Line or Bar visualization

## Appearance

The Appearance menu allows you to customize the look of your report and is primarily used in Line Chart mode. From the appearance menu you're able to:

- Show or hide the rank of your breakdown values if you have a breakdown on your report
- Set the Line Style as Straight or Smoothed
- Set the Dot Style in your line chart as Hidden, Filled, or Outlined
- Show or hide X grid lines
- Show or hide Y grid lines
- Set the grid lines as dashed or solid lines

![](/images/docs/259b21b7cd63231705bf1915c648b7e8c4722b39081dd845e995aaa2f24fdd17-appearance.gif)

## Left Y-Axis

The Left Y-Axis options allow you to format the value and appearance of your Y-Axis on your report. From the Left Y-Axis menu you're able to:

- Show or Hide the Axis values
- Add a custom Label to your Axis, so if you're measuring 'Sum of Revenue' you can rename it to 'Total Revenue' and better align with how your organization names those metrics.
- Apply formatting to the Axis which allows you to display the Axis as a number (#), financial value (\$), or percentage (%). You're also able to round decimals to the proper place value.
- Truncate the Axis with our 'Bound' option that allows you to set an upper and lower limit on the Axis. This is useful if you don't want the Axis to start at 0 or you have an upper threshold that should be respected by the results.
- Show Reference Line allows you to set a static line on your report. Reference lines are great for goals or benchmarks that you want to be visible at all times by all users. Adding a reference line let's you quickly see if a data point is above or below the threshold.

![](/images/docs/f8c03fa49634cefa076bdb44fbc0186ad8ea7ad11fb371cbaa92f6bac1abb0a3-Feb-10-2026_18-03-24.gif)

## X-Axis

The X-Axis menu allows you to apply formatting on the X-Axis. From the X-Axis menu you're able to:

- Show or Hide the Axis values
- Add a custom Label to your Axis, so if you're measuring 'Countries in Europe' you can rename it and better align with how your organization names those groups.

## Sorting

The Sorting menu allows you to control the sorting of measure and breakdown values. Within the Sorting menu you're able to nest the sorting and based on the order of the item the sort will be nested.

When creating a Query report with multiple measures or breakdowns and segments, it can be beneficial to sort these results based on various values. Kubit's sorting option allow you to:

- Sort results Alphabetically, ascending and descending
- Sort results by Average, ascending and descending

Sorting is currently only available in Query but we plan on expanding this to other report types soon. Kubit's sorting functionality will keep the sort within each breakdown group, with the first group being the "parent" and second group as the "child".

How to enable Sorting for your report results:

1. Create a Query report
2. Click the `Chart Settings` tab on top of the Query Builder section
3. Expand the Sorting section of the settings menu
4. Select the sorting logic you want to use for your report. You're able to sort by Measures as well as discreet breakdown groups.
5. Click Save to save your report settings.

![](/images/docs/d68482eedb064001aeab35cdede93bc0f621eb2c9c305a5a97c2bcbc9deeb9d7-sorting.png)

# Compare Multiple Measures on a Chart

You can compare up to 6 measures in a chart by de-selecting the Compound Measure and selecting the desired Measures via the colored legend `Measure Selectors` below the chart.

![](/images/docs/cf9c96617a1803ef30c7b8dab916a1f78e5f0ff669d3e2b95dd22098b12ce377-compare_multiple_measures_on_a_chart.gif)

# Rename or Format Measures on Chart

Once you've constructed your chart the naming in the Measures legend should be easy to understand. To rename them select the context menu and perform a number of formatting actions.

You're able to:

- Rename the Measure.
- Format to a percentage` %`, a number `#`, or a currency `$`.
- Convert to a different metric if the value displayed isn't ideal.
  - An example could be you've got a Measure with a Sum of Seconds Watched and you want to convert it to Hours.
- Create a [Cohort](doc:cohorts) of all users within the Measure result.

![](/images/docs/339eec75c09a23b76483f4d649c26e6e7c0874caeb34cc009adf504db47265ca-rename_or_format_measures.gif)

# Data Table

At the bottom an executed report, there is a data table.

![](/images/docs/cb38de9838aa924f37cc867eb200b98d2f81c9de51b4627eb43cb709c9a9b537-Data_Table.png)

This will contain one or more rows. The number of rows seen will depend on the number of measures chosen, or breakdowns selected, and so on. If you have a simple report with just one measure, you'll only have one row.

The columns from left to right are as follows: Rank, Measure, Breakdown field (if you selected more than one), the Average of the values, followed by the dates in ascending order.

![](/images/docs/030de63efb4620dc08d18d548bfa136397d675e1837cefed983ab30359ef1010-image.png)

At the top of the Data Table section, there are options to display the top X rows, Export to CSV, Filter on a value, or show Horizontal bars.

![](/images/docs/e2d014323c188173552cc63450fc5f0cc4139df07e3c525d79f07f0f7f0c5ac2-image.png)
