# What is AI Readiness?

Lumos AI Readiness is a diagnostic scoring feature designed to evaluate the health and accuracy of your data model for Kubit’s Agentic Chat Experience. It ensures that your underlying data structures, naming conventions, and event mappings are optimized so that Kubit's AI Agents can provide accurate, trustworthy insights.

AI Readiness must understand the intent behind the data, ensuring your team can trust the automated narratives and diagnostic reports generated by Kubit.

The readiness chip appears on the Lumos AI Chat and Data Dictionary screens so you always know your current status:

![](/images/docs/fbc1e3abe74150adbc6992e7502a8a463fa907268248c95b2424a47cfa6cabfd-Feb-19-2026_12-12-17.gif) ![](/images/docs/71b45b76ea0c82d640d9bdaa18ca469d79a7a7ff912357c275dacf59f27c7020-Feb-19-2026_12-15-06.gif)

# Schema Definition

This pillar focuses on the core structure of your data models.

Clear schema definitions prevent the AI from missing valuable information during its reasoning process.

![](/images/docs/728210f90c3d7ebf4e4341f3ec547ccfdffdde9b19cc949721a4b024de480e95-Screenshot_2026-02-19_at_12.23.42.png)

When you are done updating you Schema Description, don't forget to click `Confirm` at the top right corner of the page to apply all changes.

# Schema Instructions

Beyond the raw structure, the AI needs specific guidance on how to use your schema.

We have provided you with a predefined company instructions template in your Kubit environment, ready for you to populate. It covers 8 key areas:

* Daily Active User (DAU)
* Key Conversion Events
* User Journey Stages
* "Aha Moment" / Value Realization
* Industry-Specific Terms
* Internal Product Terminology
* Acronyms & Abbreviations
* Key Definitions

<Info>
  ### Instruction Tips:

  • Write one instruction per line, using quotes to specify exact names or phrases (e.g. Use "completed\_purchase" event for any revenue analysis). • Format instructions as direct statements of the desired behavior—be specific! • Define key metrics using your event and field names to ensure accuracy. (e.g. an active user is defined as subject = "User" and event name = "login" or "sign up").
</Info>

<Warning>
  Please make sure to **ONLY** populate the existing Schema Instruction template **NOT** replacing it with free text. Any additional information aside of the default 8 topics can be added at the end of the template and will be used for AI's reasoning analysis.
</Warning>

Providing custom instructions of your Schema are key components on defining join logic for AI Kubit Agents.

You can navigate to the "Schema Instructions" section either through the **Setting Menu > AI Settings** on the left left-hand side navigation or through clicking on the AI readiness chip directly:

![](/images/docs/ee30198e3355c84b10347ecca414a36d822ca999c61e38ff8ccc882847fdd593-Feb-19-2026_12-31-54.gif)

When you are done updating you Schema Instructions, don't forget to click `Save` at the top right corner of the page to apply all changes.

# Events

For behavioral analytics, the AI must understand what each user action represents. This pillar tracks the documentation and naming of your Events. Ensure every event has a human-readable description and proper tag categorization.

We have now Kubit’s AI Data Dictionary feature which streamlines your workflow by automatically suggesting descriptions and tags, significantly reducing the time spent on manual documentation.

**How to Use It**

* *Access the Dictionary*: Click on the Dictionary icon in the sidebar and navigate to the “Events” tab or click on the "Events" tile directly from the AI Readiness Chip

* *Generate Metadata*: For any event missing a description or tag click the "AI Suggest" icon:

![](/images/docs/695d62199a8d0c341ee23581b48a754123c81fb0ba885a30250f7e02bbdbe309-image.png)

to see a proposed definition based on the metrics SQL logic, Event name, table name and key.

* *Review*: Make sure you review the AI output and reflect any changes if necessary.

You can also use the AI Data Dictionary to add description and tags across several events at the same time:

![](/images/docs/fb7bfe32e13e6cc58d61450c5f74dcf25cd3775bccf7ab2a66c41b45cbc80a19-Feb-19-2026_12-52-01.gif)

When you are done modifying your Events details, don't forget to click `Confirm` to apply all changes.

When an event gets its description, the system will reflect it as `AI Ready` and reflect it in the AI Readiness Chip:

![](/images/docs/0489bc2d2f29e6317446d3ae380225c4a232275323cff396e4142a82efb7e158-image.png)

# Understanding Your AI Readiness Score

**Progress Percentage:** The central dial represents your overall AI Readiness score. As you complete items in each category, this percentage will increase.

**Task Tracking:** Each pillar displays the number of remaining tasks required to reach 100% readiness.

**Continuous Improvement:** AI Readiness is not a one-time setup; as your data evolves, you should revisit these pillars to maintain high-fidelity analysis.
